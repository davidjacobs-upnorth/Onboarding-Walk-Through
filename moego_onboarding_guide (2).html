Sprint 0 — Pre-Setup

Remove all friction before the first real session. If data isn't ready here, every sprint after it gets delayed.

◎ Goal: Remove friction before kickoff

Checklist

✦ Data intake (clients, pets, services)
✦ Payment setup (critical path)
✦ Basic config template applied
✦ Define success criteria + MAP (Mutual Action Plan)

Salesforce Fields

Data Received ✓ Payments Status Integration Required?

Exit Criteria

Kickoff scheduled
Data ready or scoped
Customer aligned on timeline
3
Sprint 1 — Foundation

Get the customer operational as fast as possible. The goal is one real booking in MoeGo.

◎ Goal: Get them operational ASAP

Checklist

✦ Calendar + services configured
✦ Staff setup
✦ Payments live
✦ First booking flow tested

Deliverable

🎯 First live booking + payment processed

Ceremonies

Kickoff call (Day 1) Daily async check-ins (Slack/SMS/email) End-of-week review

Salesforce Fields

Calendar Config ✓ Staff Setup ✓ Test Booking ✓
→ Exit when: First booking completed in MoeGo
4
Sprint 2 — Activation

Replace the old workflow. Customer should be running day-to-day inside MoeGo — not alongside it.

◎ Goal: Drive real usage + replace old workflows

Checklist

✦ Online booking live
✦ SMS reminders + confirmations
✦ Basic reporting introduced
✦ Staff trained on daily workflows

Deliverables

📊 % of bookings happening inside MoeGo
⚡ Automation turned on

Salesforce Fields

Online Booking ✓ SMS Automation ✓ % Bookings in MoeGo
→ Exit when: Customer is actively using the system for live bookings
5
Sprint 3 — Expansion

Unlock revenue features. This is where MoeGo starts paying for itself in the customer's mind.

◎ Goal: Unlock revenue levers

Checklist

✦ Memberships / packages
✦ Upsells & add-ons
✦ Review generation
✦ Abandoned booking recovery

Deliverable

📈 Measurable ROI signal — more bookings, less no-shows, higher average ticket

Salesforce Fields

Packages Enabled ✓ Upsells Enabled ✓ Review Automation ✓
→ Exit when: At least one revenue feature adopted
6
Go-Live (Revenue Active)

Customer is fully live. Payments flowing, bookings happening, automation running.

Key Fields

First Payment Date Weekly Booking Vol. Adoption Score
7
Sprint 4 — Optimization & Handoff

Transition to the long-term CS team. Document health and performance so nothing is lost in handoff.

◎ Goal: Transition to long-term success

Checklist

✦ Advanced workflows configured
✦ KPI dashboard review completed
✦ Ownership transfer to CSM / support

Salesforce Fields

Time to Go-Live Health Score CSM Assigned
→ Exit when: CSM accepted and first check-in scheduled

02 · Core Metrics

Salesforce Formulas

These three formulas separate a support ticket tracker from a real revenue pipeline. Build them into your record layout so they're always visible.

Time to Go-Live
How long did it take from deal close to the customer going live? Your core efficiency metric.
Go_Live_Date − Close_Date
Time to First Value (TTFV)
When did the customer make their first booking? Earlier = less churn risk.
First_Booking_Date − Close_Date
Adoption Score (v1)
A simple 0–1 score. Are the three core features live? Each one is worth ⅓.
( Online_Booking + SMS_Auto + Payments_Live ) / 3
Health Score Weights
Payments gets the most weight — it's the clearest signal that MoeGo is truly running the business.
